## Recommended learning order

1. Run the naive and seasonal-naive baselines.
2. Inspect chronological train/test splitting.
3. Run the complete open-dataset benchmark.
4. Compare per-dataset results instead of trusting only the overall rank.
5. Re-run the benchmark with a different holdout horizon.
6. Add a new model only after confirming it beats a simple baseline on at least one appropriate dataset.

## Documentation standard

Maintained documentation should be executable, specific and honest. Commands must match the current package, benchmark claims must be reproducible, and limitations must be stated alongside results.
