# Support

## Before asking for help

1. Read the [documentation hub](docs/README.md).
2. Check the [FAQ](docs/FAQ.md) and [troubleshooting guide](docs/TROUBLESHOOTING.md).
3. Search existing issues.
4. Reproduce the problem with the smallest possible public or synthetic dataset.
5. Record Python, operating-system and dependency versions.

## Where to ask

- Use a bug report for reproducible defects in maintained package code, tests or benchmark execution.
- Use a model proposal for a new algorithm, dataset, metric or evaluation change.
- Use pull requests for focused, tested improvements.
- Follow `SECURITY.md` for vulnerabilities or accidental exposure of sensitive information.

## Scope

Community support focuses on the maintained package, benchmark, tests and current documentation. Historical notebooks are available for learning but may depend on obsolete notebook runtimes or library versions.

## Good support requests

A useful request includes:

- exact command or minimal code;
- expected and actual behaviour;
- complete relevant error output;
- Python and package versions;
- public or synthetic reproduction data;
- confirmation that temporal order was preserved.

Do not attach payment, health, identity, authentication or other confidential data.
